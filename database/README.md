# KeySwift 数据库构建目录

本目录保存当前开发阶段的数据库显式构建文件。

## 目录职责

- `bootstrap/`：启动配置库 schema，负责保存主库连接、端口和本地加密密钥等启动前配置。
- `main/`：主业务库 schema，负责保存宿主当前运行表、插件治理表和插件数据库声明表。
- `plugins/`：插件数据库声明规则说明，不保存具体插件的业务 schema。

## 执行规则

- 普通服务首次启动时，如果配置库或主业务 SQLite 文件不存在，会按当前基线创建干净数据库。
- 如果数据库文件已存在，普通服务只校验结构元数据，不补字段、不执行自动迁移。
- 已存在数据库缺少元数据或版本不匹配时，删除对应数据库文件后重启，由程序按当前基线重新创建。
- 需要手动重建全新开发数据库时，也可以使用 `go run ./cmd/dbctl -target all`。
- 数据库结构变化必须同步更新本目录 schema、模型、仓储、服务和文档。
- 当前开发阶段不维护历史迁移链，不保留旧字段兼容逻辑。

## 当前基线

- SQLite 是当前必须可用的宿主基线。
- MySQL 和 PostgreSQL 后续按相同结构补齐方言文件。
- 当前 schema 版本：`2026.06.20.1`。
