# 06 开发调试与验收

## 1. 本地开发流程

推荐流程：

1. 新建插件目录。
2. 编写 `manifest.json`。
3. 编写后端入口或前端资源。
4. 编写数据库声明。
5. 编写 `checksums.json`。
6. 放入 `plugins/<plugin_id>/releases/<version>`。
7. 在后台或接口执行插件刷新。
8. 查看插件详情、权限、绑定和数据库声明。
9. 显式执行数据库结构操作。
10. 启用插件并验证主链路。

## 2. 数据库初始化

宿主数据库初始化命令：

```bash
go run ./cmd/dbctl -target all
```

可选参数：

```bash
go run ./cmd/dbctl -target all -config-dir user_config -main-db user_config/user_data.db -server-port 8080
```

规则：

- 普通服务启动不会自动创建表。
- 缺少 schema 元数据时服务启动失败。
- 修改 schema 后必须重新构建开发数据库或执行明确的一次性结构处理。

## 3. 插件刷新检查

刷新后检查：

- 插件是否出现在插件列表。
- 插件状态是否为已发现或已禁用。
- manifest 是否有解析错误。
- 权限是否进入权限列表。
- 菜单和页面是否进入绑定列表。
- 数据库表声明是否进入 `plugin_database_tables`。
- 字段、索引、关系是否完整落库。

## 4. 插件验收清单

| 项目 | 标准 |
| --- | --- |
| manifest | 必填字段完整，版本和平台声明正确 |
| 完整性 | `checksums.json` 存在，hash 可校验 |
| 权限 | 所有页面、配置、动作都有权限点 |
| 数据库 | 表名、字段、索引、关系符合规范 |
| 安全 | 敏感字段已标记，日志不泄露密钥 |
| 后端 | 路由、事件、任务有超时、幂等和错误处理 |
| 前端 | 页面可挂载，加载态、空态、错误态完整 |
| 审计 | 高危操作写入审计 |
| 禁用 | 插件停用后不再贡献页面、菜单和流量 |

## 5. 支付插件专项验收

- 支付配置可保存和隐藏敏感字段。
- 支付拉起成功。
- 回调可验签。
- 重复回调不会重复确认。
- 金额不一致时拒绝。
- 订单不存在时拒绝。
- 支付事实只提交给宿主，由宿主裁决订单状态。

## 6. 发卡插件专项验收

- 卡密导入可去重。
- 卡密库存可锁定。
- 并发发卡不会重复发同一张卡。
- 发卡成功后提交交付事实。
- 发卡失败后任务可重试。
- 卡密查看、导出和删除有权限与审计。

## 7. 主题插件专项验收

- 主题 token 可加载。
- 后台页面可应用主题。
- 用户端和后台作用域隔离。
- 主题停用后恢复宿主默认样式。
- 主题插件不写业务表和不修改业务状态。

## 8. 交付文件清单

插件交付至少包含：

- `manifest.json`
- `checksums.json`
- 后端二进制或服务入口。
- 前端 bundle 或 iframe 入口。
- 数据库声明。
- 配置 schema。
- 插件说明文档。
- 验收记录。
