# 05 安全与审计规范

## 1. 完整性校验

插件必须提供 `checksums.json`。

启用前宿主会检查：

- manifest 是否可解析。
- 当前平台是否支持。
- checksum 文件是否存在。
- 数据库声明是否合规。
- 权限声明是否合规。

后续接入签名后，还应校验：

- 包签名。
- manifest 签名。
- 二进制签名。
- 审批指纹。

## 2. 数据库安全

插件不得直接获取主库 DSN。

原因：

- SQLite 无法对插件做表级隔离。
- 直接 DSN 会绕过宿主审计。
- 插件可能误改宿主核心表。
- 插件可能读取其他插件敏感数据。

插件数据访问必须通过宿主受控能力。

## 3. 敏感字段

以下字段必须声明为敏感：

- 支付密钥。
- Webhook 密钥。
- 第三方平台 token。
- 卡密明文。
- 私钥。
- 证书密码。
- 外部 API 凭据。

敏感字段规则：

- 不允许明文返回给普通列表接口。
- 必须审计查看、导出和修改。
- 配置类敏感字段应使用密钥引用或加密保存。
- 日志中不得输出敏感值。

## 4. 高危权限

以下操作属于高危：

- 启用或停用插件。
- 执行插件数据库结构操作。
- 修改支付渠道配置。
- 导出卡密。
- 查看密钥或敏感配置。
- 发起退款或对账修正。
- 批量删除插件数据。

高危权限必须有独立权限点。

## 5. 审计事件

必须审计：

- 插件发现失败。
- 插件启用、停用。
- 插件完整性校验失败。
- 插件数据库声明变更。
- 插件数据库结构执行。
- 插件配置变更。
- 插件敏感数据读取。
- 插件导入导出。
- 插件向宿主提交支付或交付事实。

## 6. 日志要求

日志应包含：

- `request_id`
- `trace_id`
- `plugin_id`
- `plugin_version`
- `operation_id`
- `resource_type`
- `resource_id`
- `status`
- `error_message`

日志不得包含：

- 密码。
- token。
- 私钥。
- 卡密明文。
- 支付密钥。
